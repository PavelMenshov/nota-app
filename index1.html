1.1 Workspace / Page (ядро)

Workspace → Pages. A page contains “surfaces”: doc, canvas, and PDF (PDF as attached file + annotations). Version history + audit trail, link sharing, and role-based access.

Page = container for surfaces

Keep UX simple: users open a page and switch surfaces (Doc / Canvas / PDFs) without navigating away. This is the “Notion feeling” but purpose-built for academic assets.

Suggested primitives: workspace_id, page_id, surface_type, object_ref

Capabilities
  • Workspace → Pages hierarchy
  • Surfaces: Doc / Canvas / PDF attachments
  • History: version snapshots + diffs
  • Audit: who changed what, when
  • Share link + roles (viewer/editor)

1.2 Real-time collaboration (Doc + “Miro”)

Collaborative editing for docs and canvas with presence (who’s online) and minimal cursors/selection. Conflict resolution via CRDT/OT.

Presence + cursors, but quiet

Don’t gamify it. Show small avatars, a subtle “live” indicator, and optional cursors. Most users want calm, not fireworks.

Implementation note: doc blocks + canvas objects can share the same collaboration layer.

Minimum spec
  • Presence: active users list
  • Optional cursors/selection
  • CRDT/OT conflict resolution
  • Offline merge-friendly writes

1.3 Files and PDF

S3-compatible object storage, upload/download, PDF annotations (highlights + ink + comments), and extraction: highlights → doc blocks.

PDF as a first-class source

Make PDF annotation reliable: highlight ranges, freehand ink, and anchored comments. Then allow “extract highlights” to instantly build a study doc page section.

UX tip: one button Extract → Doc creates structured blocks with citations.

Annotation types
  • Highlight (text ranges)
  • Ink (stylus strokes)
  • Comments (anchored)
  • Export highlights → doc blocks

1.4 Export / send results

Export Page → PDF and DOCX (server-side job). For B2C: “Send to Notion” — create a page with content/attachment.

Exports as background jobs

Generate PDF/DOCX server-side to keep the client clean. Provide a single “Export” entry point on the page with format choices.

B2C bridge: Send to Notion creates a page + attaches exported file.

Deliverables
  • Export PDF (print-ready)
  • Export DOCX (editable)
  • Job status + download link
  • Send to Notion (page + file)

1.5 Tasks + Calendar (minimum)

Tasks: title, due, status, links to pages. Calendar: events. In B2B: sync schedules/exams/deadlines from LMS.

Tasks and time: the “anti-procrastination” layer

The simplest implementation wins: tasks live next to pages and can create calendar events. B2B integration can come later via LMS sync.

Keep schema tight: task → page_id, event → task_id?

Minimum schema
  • Task: title, due, status, page links
  • Calendar: events + reminders
  • B2B: LMS sync (deadlines/exams)
